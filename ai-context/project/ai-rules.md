# AI Rules

Before changing code:

1. Read architecture.md
2. Read mistakes-to-avoid.md
3. Read current-sprint.md

Never:
- Rebuild working components.
- Replace working authentication.
- Change database schema without approval.
- Delete code to fix errors.
- Assume a component is broken without checking logs.

When debugging:
- Identify root cause first.
- Explain findings before rewriting code.
- Suggest minimal changes before major refactors.
